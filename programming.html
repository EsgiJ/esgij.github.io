Luden Engine

Game Engine C++ ECS ImGui Box2D SFML

What is Luden Engine?

Luden Engine is a lightweight 2D game engine I built from scratch in C++. It features a custom Entity-Component System (ECS), an ImGui-powered editor inspired by Unity and Unreal, Box2D 3.0 physics integration, and a hot-reload capable native scripting system.

This project holds a special place in my heart. When I started, I was just a guy with little C++ experience who had no idea how game engines actually worked. I didn't know how to design systems, manage memory efficiently or create editor tools. Building Luden taught me all of that and more.

Entity-Component System (ECS)

At the core of Luden is a custom ECS designed for speed and flexibility. ECS uses composition. An entity is just a lightweight ID, and you attach components to define its behavior.

For example you don't have to design your objects/entities in compile time. Compile time hierarchy is not a good idea for a realtime application like a game engine. You simply compose:

  • Player: Transform + Sprite + RigidBody + Health + Input
  • Enemy: Transform + Sprite + RigidBody + Health + AI
  • Flying Enemy: Transform + Sprite + Health + AI + Flying (no RigidBody!)

The key thing is memory layout. In OOP, each entity is scattered across memory, causing cache misses which is too expensive. In ECS all components of the same type are stored contiguously. When iterating over thousands of entities, this cache-friendly layout makes a huge difference.

Engine Architecture

Luden follows a three-layer modular architecture:

  • Engine Core (Engine.dll): ECS, physics, rendering, resource management, the shared library that everything depends on
  • Editor Layer (Editor.exe): ImGui-based visual editor with scene hierarchy, inspector, viewport, resource browser
  • Game Module (GameModule.dll): Your game scripts compiled separately and hot-reloadable

Dependencies flow one way: Editor and GameModule depend on Engine, but Engine remains independent. This means you can reuse the engine across different projects.

Editor Features

The editor includes multiple specialized tabs:

  • Scene Editor: Viewport with gizmos (move/rotate/scale), play/edit toggle, physics debug visualization
  • Sprite Editor: Define sprites from texture atlases with grid helpers
  • Animation Editor: Frame-by-frame animation with timeline, preview, and loop settings
  • Prefab Editor: Create reusable entity templates that auto-update all instances

Hot-Reload System

One feature I'm particularly proud of is hot-reload. When you modify your game scripts and recompile, the editor detects the DLL change and:

  • Destroys all script instances
  • Unloads the old DLL
  • Loads the new DLL
  • Eegisters scripts and recreates instances again

Other Systems

  • Physics: Box2D 3.0 integration with static/dynamic/kinematic bodies, box and circle colliders
  • Animation: Frame-based sprite animation with configurable durations and loop settings
  • Input: Context-based input system with action abstraction (inspired by Unreal's Enhanced Input)
  • Prefabs: Entity templates with nested prefab support - change the prefab, all instances update
  • Serialization: JSON-based scene format for version control friendliness

Coop TPS Base Defense

Unreal Engine C++ Multiplayer Motion Matching

Overview

The inspiration for developing this game came after playing Lethal Company. Its engaging co-op gameplay and the thrill of working together with friends to survive sparked a desire to create my own cooperative experience. I wanted to design a game that emphasizes teamwork, strategy, and the excitement of defending a base against relentless waves of enemies.

Gameplay Video

Gameplay Mechanics

Players collaborate to defend designated entry points of the base against waves of zombies. Each player shares the same abilities, focusing on shooting and resource management. Communication and teamwork are vital for surviving increasingly challenging assaults.

Technical Features

  • Networking: Server-client architecture with state replication and synchronization
  • Motion Matching: Smooth, realistic character animations using chooser tables
  • AI: Behavior Trees for complex zombie decision-making and pathfinding
  • PCG: Procedural Content Generation for forest environment
  • Metahuman & Nanite: High-quality character models with efficient rendering

Motion Matching

Weapons

The game features four different weapons: a pistol, shotgun, rifle, and grenade launcher. Each weapon has distinct ammo counts, damage values, area effects, and bullet spread rates.
